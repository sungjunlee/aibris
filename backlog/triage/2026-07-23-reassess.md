# Backlog Reassessment — 2026-07-23

## Evidence

- Milestones #3-#5 and the implementation sprint for milestone #6 are
  complete. The default path now preserves all-category visibility, invalid
  selectors and partial scans fail truthfully, and compiled CLI contracts
  protect the behavior.
- The remaining UX debt is structural: `cmd/clean.go` still coordinates a
  guided worktree run and a classic cleanup run as two plans, confirmations,
  and receipts. Their overlap is reconciled only after guided review.
- Worktree policy already has deterministic decisions and stable reason codes,
  while classic cleanup has filtering and audit accounting but no shared
  selection model.
- Release gate #111 remains intentionally open. Milestones #7-#9 have no due
  dates, and the roadmap explicitly keeps the project in 0.x until maintainer
  approval.

## Assessment

Milestone #7 remains the correct next initiative. A shared cleanup-plan model
is the dependency for both a coherent review surface and a single execution
contract; starting with rendering or command rewiring would duplicate policy
and accounting. Distribution and automation/schema work should remain queued
behind this UX foundation.

## Decision

- Keep the existing no-date roadmap and issue scope unchanged.
- Execute milestone #7 in dependency order: model (#113), renderer (#114),
  execution contract (#115), dogfood (#116), epic verification (#112), then
  the maintainer-controlled release gate (#117).
- Treat #117 like #111: it may outlive the implementation sprint and must not
  create a tag, date, or v1.0.0 implication without explicit approval.
